# Post-Hackathon Roadmap

## Phase 1 — Production hardening (0–6 weeks)

- Complete owner-supervised read-only Excel adapter validation.
- Add formal golden datasets, prompt evaluations and regression automation.
- Add encrypted transport, enterprise secret management and retention controls.
- Add field-level policy configuration and privacy review.
- Add latency, cost and output-quality dashboards without financial payload logging.
- Conduct cross-organisation collision and high-volume performance tests.

## Phase 2 — Desktop evolution (6–12 weeks)

- Package the intelligence service as a signed local or managed desktop companion.
- Add permission-aware Copilot entry points in the Treasury Management Platform.
- Add management-summary export while keeping ledger data read-only.
- Add approved-reconciliation correction explanations based on the future reversal workflow.

## Phase 3 — Multi-platform adapters (3–6 months)

- Google Sheets / Apps Script read-only adapter.
- Web client with enterprise identity and tenant policy enforcement.
- Android and iOS clients consuming the same service contracts.
- Cloud API with tenant isolation, key management, audit metadata and regional deployment controls.

## Phase 4 — Advanced intelligence (6–12 months)

- Evaluated, human-reviewed match recommendations with calibrated confidence.
- Exception clustering and recurring-cause analysis.
- Cash-position explanations and executive treasury reporting.
- Policy and control monitoring.

## Non-negotiable future write boundary

Any future action must call the Treasury Management Platform’s validation and authorization services, enforce existing permissions and maker-checker rules, create audit evidence, support user confirmation, and remain separately approved. Direct model writes to workbook tables are prohibited.
